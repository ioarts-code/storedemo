📋 SETUP IN 3 STEPS
═══════════════════════════════════════════════════════════════════════════════

1️⃣  HYGRAPH SCHEMA
   Create these models in your Hygraph project:
   
   Service:
   ├─ name (Text, required)
   ├─ description (Rich Text)
   ├─ shortDescription (Text, optional)
   ├─ url (URL, optional)
   ├─ icon (Text, optional)
   ├─ image (Asset, optional)
   ├─ category (Reference → Category)
   └─ tags (Multiple Reference → Tag)
   
   Category:
   ├─ name (Text, required)
   └─ slug (Slug)
   
   Tag:
   ├─ name (Text, required)
   └─ slug (Slug)

2️⃣  GET API ENDPOINT
   ├─ Log in to hygraph.com
   ├─ Open your project
   ├─ Go to API Access → Endpoints
   ├─ Copy the "Content API" endpoint
   └─ (Optional) Create an Auth Token in API Access → Tokens

3️⃣  CONFIGURE APP
   ├─ Start the app: pnpm dev
   ├─ Click the ⚙️ icon in top-right
   ├─ Paste your API endpoint
   ├─ Optionally add your auth token
   ├─ Click "Save Config"
   └─ Services will load automatically!

═══════════════════════════════════════════════════════════════════════════════

🎯 API ENDPOINT FORMAT
═══════════════════════════════════════════════════════════════════════════════

https://api-[region].hygraph.com/content/[project-id]/[stage]

Example:
https://api-eu-central-1.hygraph.com/content/c1a2b3d4/published

✓ Look in Hygraph dashboard under: API Access → Endpoints → Content API

🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

"Invalid endpoint" error
└─ Check URL format: https://api-[region].hygraph.com/content/[id]/[stage]
└─ Test in Hygraph GraphQL Playground

Services not loading
└─ Verify services are Published (not Draft) in Hygraph
└─ Check all required fields are populated
└─ Look at browser console (F12) for GraphQL errors

Auth token issues
└─ Try removing token and using public access
└─ Verify token hasn't expired in Hygraph
└─ Check token has correct permissions
